Entendendo Comunicação Client x Server

(Voltar)

Ao acessar um site, ocorre uma interação entre o lado Client e o lado Server, o que define a experiência do usuário. Quando um usuário deseja acessar um site, o navegador solicita ao servidor arquivos, que podem ser HTML, CSS, JavaScript, imagens, áudios e etc. É essencial adicionar que a velocidade de conexão e tamanho dos arquivos afetará o carregamento da página. O navegador gera cache, ou seja, armazena na máquina Client os arquivos, o que permite que futuros acessos sejam mais rápidos. Usando F5, ou Refresh, se limpa o cache e atualiza os arquivos diretamente do servidor.

Aspectos muito importantes em relação a navegadores e o design de arquivos são a compatibilidade entre browsers e um design responsivo (design que se adequa a diferentes dispositivos e telas).

Os servidores desempenham um papel fundamental na disponibilidade e funcionamento de aplicações web. Eles não só armazenam e servem os arquivos estáticos, mas também podem se conectar a bancos de dados, fornecendo dados dinâmicos conforme necessário. As aplicações web, ao contrário dos aplicativos instalados, funcionam através do navegador web e aproveitam a internet como meio de comunicação entre cliente e servidor. Existem servidores de arquivos, servidores de segurança, servidores de streaming, servidores de email, servidores Web. Os servidores são baseados em 2 partes: softwares e hardwares. Softwares são sistemas operacionais, servidores Web, monitoramento. Alguns exemplos são Windows Server, Ubuntu, Oracle. Apache e Nginx são exemplos de servidores Web. Hardwares englobam armazenamento (disco rígido, SSDs), memória, processadores.

Linguagens de Programação e Client x Server

HTML, abreviação de Hypertext Markup Language, não se enquadra na categoria de linguagens de programação, mas sim como um arquivo de marcação. Enquanto linguagens como PHP, Ruby, Java, C# operam no lado do servidor, manipulando dados e executando operações, o HTML fornece uma estrutura para apresentar conteúdo na web.

Introdução ao HTML na Prática

(Voltar)

A estrutura básica do HTML compreende três elementos fundamentais: html, head e body. O conteúdo entre as tags body é o que se apresenta diretamente ao usuário, enquanto o head contém os elementos que são processados antes da exibição. A tag title dentro do head especifica o título da página. Os atributos das tags oferecem uma variedade de funções para personalizar os elementos HTML. O atributo id permite identificar de forma única um elemento dentro do documento, enquanto o style é utilizado em conjunto com CSS para modificação de estilos de formatação. Pode-se mencionar também o atributo type, específico para determinar o tipo de entrada, como texto e número.

Em relação à formatação do texto em HTML, tem-se disponível várias tags com diferentes finalidades. Tags como h são usados para cabeçalhos e títulos, p é utilizado para parágrafos, enquanto blockquote é utilizado para citações. Outras tags como mark para destaque e small para textos em letras menores também são utilizadas para maior diversidade visual do conteúdo.

Pode-se mencionar também as listas ordenadas e as listas não ordenadas. As ordenadas não aceitam texto comum; necessitam de tags filhas. Se utiliza a tag ol para ordenadas e ul para não ordenadas.
